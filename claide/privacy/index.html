Privacy Policy

Effective: February 2026

Summary

Claide (com.hexul.claide) processes and stores all data locally on your Mac. It does not collect, transmit, or share any user data.

Data Collection

Claide does not collect any data from its users. The app does not use analytics, telemetry, crash reporting, advertising frameworks, or any tracking technology.

Terminal Sessions

Claide runs terminal sessions locally using zsh. All commands and output remain on your device and are never transmitted externally.

Issue and Task Data

Claide reads issue and task data from local files, including beads CLI output and Claude Code task files. This data is read from your local filesystem and is never sent to any external service.

File Change Monitoring

Claide monitors local session files for changes to provide real-time updates. This monitoring is performed entirely on your device.

Session Transcripts

Claide reads Claude Code JSONL session transcript files stored locally on your Mac. These files are read from disk and are never uploaded or transmitted.

Software Updates

Claide uses the Sparkle framework to check for application updates. When checking for updates, Sparkle sends your macOS version and app version to the update server. No personal data, device identifiers, or usage information is included in these requests.

Network Usage

Apart from Sparkle update checks, Claide does not access the network. It makes no other HTTP requests, has no server component, and does not sync data to any cloud service.

Tracking

Claide does not track users in any way. The app does not use cookies, device identifiers, or any form of tracking technology.

Third-Party Services

Claide does not integrate with any third-party services or SDKs beyond the Sparkle update framework described above.

Children's Privacy

Claide does not collect data from anyone, including children.

Changes

If this policy changes, the updated version will be posted at this URL.

Contact

Questions about this policy can be directed to hexul.com.
